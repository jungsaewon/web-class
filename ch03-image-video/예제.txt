웹 표준(Web Standards)이란?
웹 표준은 웹을 만드는 기술의 공통 규칙으로, 모든 브라우저와 기기에서 동일한 결과를 보여줄 수 있도록 정해진 국제적인 약속입니다. 대표적인 예로 HTML, CSS, JavaScript, DOM, WAI-ARIA 등이 있습니다.

웹 표준은 W3C(World Wide Web Consortium)와 WHATWG 같은 국제 단체가 주도하여 제정합니다. 웹 표준을 지키면 호환성, 접근성, 유지보수성이 모두 향상됩니다.

1. 웹 표준의 필요성
웹 표준은 단순한 규칙이 아니라 모든 사용자가 동일한 경험을 할 수 있게 하는 기반입니다. 다음과 같은 이유로 꼭 필요합니다.

1) 브라우저 호환성 확보 - 크롬, 사파리, 파이어폭스, 엣지 등에서 동일하게 동작
2) 다양한 기기 지원 – PC, 태블릿, 모바일, TV 등 다양한 환경 대응
3) 유지보수 용이 – 구조화된 코드로 수정과 확장이 쉬움
4) SEO(검색 엔진 최적화) – 검색 노출과 접근성이 향상됨

2. 웹 표준의 구성 요소
웹 표준은 여러 기술이 결합된 개념입니다.

HTML – 콘텐츠의 구조를 정의
CSS – 디자인과 레이아웃을 담당
JavaScript – 상호작용과 동작 구현
DOM – 문서를 객체 모델로 표현해 조작 가능하게 함
WAI-ARIA – 보조기기 사용자 접근성 지원

[다음: 웹 표준의 장점으로 이동]
3. 웹 표준의 장점
“웹의 힘은 보편성에 있다. 장애에 관계없이 모든 사람이 접근할 수 있어야 한다.” – 팀 버너스-리
웹 표준을 준수하면 다음과 같은 장점이 있습니다.

1) 호환성 – 브라우저/OS 차이 없이 동일한 결과
2) 유지보수성 – 읽기 쉽고 재사용 가능한 코드 구조
3) 접근성 – 장애인·고령자 등 다양한 사용자 지원
4) SEO 효과 – 검색 엔진 친화적인 구조
5) 확장성 – 새로운 기술 적용 용이

[다음: 웹 표준을 지키는 방법으로 이동]
4. 웹 표준을 지키는 방법
웹 표준을 준수하려면 다음과 같은 방법을 지켜야 합니다.

- 의미 있는 시맨틱 태그 사용 (제목에는 h1~h3, 버튼에는 button 등)
- HTML/CSS 코드 유효성 검사 (W3C Validator)
- 반응형 디자인 적용(Media Query)
- 이미지 대체 텍스트와 라벨 제공
- 비표준 속성 남용 금지
